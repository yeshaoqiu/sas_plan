# 家庭学习陪跑系统 — 设计文档

- 日期：2026-06-29
- 状态：已确认，待编写实现计划

## 1. 背景与目标

家中有两个孩子，分别读一年级和二年级。已知学习短板：

1. 比较惧怕写字，写字容易分心；
2. 看图写话能力较弱；
3. 数学容易犯小错（粗心）。

期望：一套**家长和孩子共同参与**的学习方法 + 软件系统，让孩子以游戏方式参与、提升兴趣和积极性；家长可每日/每周持续跟进；系统持续记录学习情况，并能自动总结、更新学习计划。

## 2. 总体定位

一句话定位：**"线下练、线上记"的家庭学习陪跑系统**。

- 实际练习（写字、看图写话、做题）仍在纸上完成；
- 家长在网页里录入学习时间、内容、结果并评分；
- 系统用游戏化激励孩子，用数据与规则帮家长持续跟进，并自动调整下一周计划。

App 的角色是**动力层 + 跟进层**，不替代纸笔练习。

### 角色与设备

- 形态：**本地网页应用**，电脑/浏览器为主。
- **家长视图**（主操作）：派发/确认任务、录入结果、打分、看报表、看系统建议。
- **孩子视图**（趣味面板）：查看积分、宠物/等级、奖励商店、兄弟排行榜——只看好玩的部分，几乎不需要打字。
- 两个孩子在同一应用内切换各自档案。

## 3. 技术栈

- **Next.js（TypeScript）+ SQLite + Tailwind CSS**。
- 一条命令本地启动，浏览器打开即用；数据存放在家里电脑的单个 SQLite 文件中。
- 零服务器成本、隐私最佳；单一语言前后端一体，便于后续维护与接入 AI。
- 备选方案（不采用）：Python(FastAPI)+SQLite+轻量页面——更朴素但游戏化动效较弱。

## 4. 核心数据模型（概念层）

具体字段在实现计划阶段细化。

- **Child（孩子）**：姓名、年级、头像、当前等级、积分余额。
- **TaskTemplate（任务模板）**：可复用练习类型，含科目、默认时长、计分规则。
- **TaskInstance（每日任务）**：某天派给某个孩子的具体任务；状态（待做/已做/已评）、实际用时、家长评分、错题数、备注、可附练习照片。
- **PointEntry（积分流水）**：每次得分/扣分/兑换记录。
- **Reward（奖励）**：奖励商店项目及所需积分。
- **WeeklyReport / 弱点标签**：系统按规则生成的趋势与建议（见第 6 节）。

## 5. 针对三个弱点的任务设计（教育内核）

### ① 怕写字、写字分心 —— 短时高频 + 专注计时

- 任务切得很小（如"认真写 5 个字"而非"写一页"），配游戏化**专注番茄钟**；
- 计分看**专注度**而非数量——中途不分心、写完即得高分并立刻奖励，降低对写字的恐惧。

### ② 看图写话弱 —— 结构化脚手架

- 每个看图写话任务给固定提问支架：**谁 / 在哪里 / 在做什么 / 怎么样 / 心情如何**；
- 孩子按支架口述或写，家长录入"用了几句话 / 是否用上支架"，逐步加分进阶。

### ③ 数学粗心 —— 检查习惯 + 错题归因

- 每道数学任务做完有一步"**检查**"动作，养成复核习惯，检查后额外加分；
- 错题按原因打标签（看错题/算错/抄错/不会），系统据此判断"粗心"还是"不会"，针对性调整下周任务量。

## 6. 游戏化机制（分三阶段）

**阶段一（MVP 必做）——积分兑换**
- 完成任务得积分/金币；专注、检查、按支架完成均有额外加分；
- "奖励商店"由家长自定义（看动画时间、零食、出游、玩具…），孩子用积分兑换；
- 最直接、家长最可控，先跑通闭环。

**阶段二——养成/升级系统**
- 每个孩子有虚拟宠物/小树，持续学习即成长、升级、解锁新形象；
- 连续打卡（streak）有额外奖励，重在坚持而非完美，对"怕写字"尤其有用。

**阶段三——兄弟排行榜 / 合作任务**
- 两人周积分榜（弱化输赢，强调各自进步）；
- 合作任务：两人都完成才解锁全家奖励，把竞争转成共赢。

## 7. 跟进与总结

### 家长跟进
- **每日**：今日任务清单 + 一键录入结果/评分；
- **每周**：自动周报——各科完成率、专注度趋势、错题原因分布、连续打卡天数。

### 规则引擎自动调计划（先行，零成本、可解释）
- 连续 3 天数学"算错"标签 ≥3 → 下周自动增加口算量并提示；
- 看图写话连续达标 → 自动提高字数/句数门槛；
- 写字专注度持续偏低 → 自动缩短单次时长、增加频次；
- 所有规则可解释、可在设置中开关与调整。

### 预留 AI 接口
- 周报数据结构在设计时即预留"总结/建议"字段；
- 后期接入 Claude API，将一周记录输入，生成更像"家庭教育顾问"的个性化总结与下周建议；
- 规则引擎为保底，AI 为增强。

## 8. MVP 范围与里程碑

- **里程碑 1（MVP）**：孩子 / 任务模板 / 每日任务录入评分 + 积分兑换商店 + 今日清单。当晚即可用。
- **里程碑 2**：周报 + 规则引擎自动调计划 + 错题归因。
- **里程碑 3**：养成/升级系统 + 连续打卡。
- **里程碑 4**：兄弟排行榜 / 合作任务 + 接入 AI 总结。

**YAGNI**：里程碑 1 不做宠物动画、不做 AI、不做排行榜，先把"派任务 → 线下做 → 录入评分 → 得分兑奖"主线跑通。
