# 管理页整理（页内子标签 Tabs）— 设计文档

- 日期：2026-07-01
- 状态：已确认，待编写实现计划

## 1. 背景与目标

管理页（`/manage`）已积累 7 个配置区块（孩子、任务模板、每日计划、奖励、加分项、评分设置、已归档），单列铺开过长、找起来费劲。用**页内子标签**归组，一次只看一组，缩短页面、理清结构。纯前端布局整理，不改数据/API/业务逻辑。

## 2. 子标签分组（5 个）

| 标签 | 包含的现有区块 |
|---|---|
| 孩子 | 孩子（增删改） |
| 任务 | 任务模板 + 每日计划 |
| 奖励 | 奖励（增删改） |
| 评分 | 加分项 + 评分设置 |
| 已归档 | 已归档（孩子/模板/奖励/加分项，可恢复） |

默认停在「孩子」标签。

## 3. 实现方式

- 仅修改 `app/manage/page.tsx`（客户端组件）：
  - 新增 `activeTab` 状态（默认 `"children"`），取值 `children | tasks | rewards | scoring | archived`。
  - 顶部加一排标签胶囊按钮（沿用主题类，当前标签高亮，风格参考顶栏导航的 active pill）。
  - 把现有 7 个 `<section>` 按上表分组，包进对应标签的条件渲染面板（`activeTab === "…" && (...)`）。
- **不改动**：所有 `useState`、`reload()`、增删改/归档/恢复 handler、`toggle` 助手、每日计划勾选、评分设置保存等逻辑原样保留；数据加载仍在 `reload()` 内一次拉全（切标签不重新请求）。
- 沿用现有童趣主题共享类（`.card`/`.btn`/`.input`/`.chip` 等）。

## 4. 约束与验证

- 不引入新依赖；不改 API/仓储/数据模型；无新增/变更测试逻辑（既有 55 项测试应保持通过）。
- 以 `npx next build` 通过 + 浏览器逐标签截图验证：五个标签各自显示对应区块，切换正常，增删改/归档/恢复等操作仍工作。

## 5. 不做（YAGNI）

- 不把 `manage/page.tsx` 拆成多个子组件文件（本次只做标签归组；文件偏大可作为后续单独的重构）。
- 不改各区块内部的表单结构或字段。
- 不做标签状态持久化（刷新回到默认「孩子」标签即可）。
